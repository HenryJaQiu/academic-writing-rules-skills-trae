# IJS / PRISMA-ScR Venue-Specific 规则

本文件是 **IJS 风格 PRISMA-ScR 范围综述的 venue-specific 叠加规则**，默认建立在 `.trae/rules/academic_base.md` 之上；这里只保留 PRISMA-ScR 范围综述与该类期刊写作特别相关的约束。

本规则基于 `other submissions/prisma-review.md` 整理，目标不是一般性的“写一篇综述”，而是产出 **可复现、可审计、可过 PRISMA-ScR 审查** 的范围综述稿件。

## 1. 综述类型与总体目标

- 论文类型默认为 **PRISMA-ScR 范围综述（scoping review）**
- 目标不是做叙述性综述，而是建立一个 **跨数据库、可复现、可审计** 的证据池
- Methods 必须明确说明检索、去重、筛选、全文获取、数据提取和趋势统计的完整流程
- 结论必须与证据池边界一致，不得把文献热度趋势写成因果结论或成熟临床证据

## 2. 检索策略统一性

- 同一主题下，必须使用 **同一关键词逻辑、同一时间窗、同一天检索**
- 允许针对不同数据库做语法适配，但 **不允许改变关键词含义**
- 不同数据库之间只允许对以下元素做最小改写：
  - 字段标签
  - 引号与截断符写法
  - 通配符适配
- 检索式必须在正文 Methods 或补充材料中给出，可直接复制复现

## 3. 时间窗与数据库适配

- 必须预先定义统一时间窗，并在所有数据库中执行一致过滤
- 若数据库 UI 只能按年份筛选，允许先按年份粗筛，再在导出记录中用 publication date 二次过滤
- 只要使用了“导出后二次过滤”，就必须在 Methods 中如实写明
- 不能把“数据库界面限制”隐藏掉，导致读者误以为所有时间过滤都在数据库内一次完成

## 4. 数据库执行与同库去重

- 每个数据库都必须单独记录：
  - 检索日期
  - 检索式原文
  - 字段范围
  - 时间窗
  - 排除文献类型
  - 初始命中数
  - 排除综述后的命中数
  - 同库去重后的命中数
  - 年度分布统计
- 同库去重规则必须显式给出，优先级建议：
  - DOI
  - PMID / 数据库唯一 ID
  - Title + Year + First author
- 对 article in press、epub ahead of print、正式发表版并存的情况，优先保留正式版或信息更完整的版本

## 5. 跨库去重与合并

- 跨库合并必须使用统一主键优先级，不可人工随意判断
- DOI 必须做规范化处理：
  - 全部转小写
  - 去掉 `doi:` 前缀
  - 去掉 `https://doi.org/` 前缀
  - 去除空格和末尾句点
- 对无 DOI 的记录，必须先做 title 规范化后再比较
- 同一工作若存在 conference 与 journal 扩展版：
  - 若内容被视为扩展关系，应保留并在数据提取中标注
  - 不可简单粗暴当成完全重复删除

## 6. 排除标准与筛选审计

- 必须明确排除综述类文献，如 Review / Systematic Review / Meta-analysis
- 全文筛选时，必须逐条记录排除理由，并可汇总到 PRISMA 流程图
- 题目级 triage、题目+摘要双人盲筛、全文筛选三步若实际执行了，必须如实区分
- 若先做“快速收敛全文池”，也要说明收敛规则与保留对照逻辑
- 不允许只给最终纳入数，不报告各阶段筛减过程

## 7. Methods 与 Results 的写法要求

- Methods 必须至少包含：
  - PRISMA-ScR 遵循声明
  - 数据库列表
  - 完整检索式或补充材料中的完整检索式
  - 时间窗
  - 去重规则
  - 纳入/排除标准
  - 筛选流程
  - 数据提取字段
- Results 必须至少包含：
  - PRISMA 流程图
  - 每库命中统计
  - 去重后纳入数量
  - 年度趋势图或表
  - 证据层级/任务类型/应用场景等结构化分布

## 8. 图表与补充材料

- PRISMA 流程图必须能追踪：
  - 初始命中
  - 去重
  - 题目/摘要筛选
  - 全文筛选
  - 最终纳入
- 年度趋势图必须与最终纳入或指定统计口径一致，不能混用“初始命中趋势”和“纳入文献趋势”而不说明
- 若使用全文缓存、PDF 文本抽取或自动化脚本，应在补充材料或方法中说明
- 检索日志、去重摘要、纳入排除表若篇幅过长，可放 Supplementary，但正文必须明确引用

## 9. 注册、复现与数据管理

- 若做范围综述注册，应优先记录 OSF / figshare / 其他可公开注册平台；若不注册，也应在文中说明
- 检索日志、导出文件、去重脚本、筛选表、全文缓存策略建议保留可审计记录
- 推荐保留以下产物：
  - fulltext manifest
  - text cache
  - cache meta
  - dedupe summary
  - screening decisions
- 若存在无法获取摘要或全文的记录，需要说明其处理方式

## 10. 常见高风险点

- 把不同数据库的不同检索式说成“相同检索”
- 没有说明导出后二次时间过滤
- 只报告总纳入数，不报告同库去重与跨库去重规则
- 把综述排除写得模糊，导致原始证据与二次综述混计
- PRISMA 流程图与正文数字对不上
- 年度趋势的统计口径与最终纳入口径不一致
- 把热门方向的增长趋势写成“临床已成熟”或“已形成共识”

## 11. 投稿前检查清单

- [ ] 综述类型明确标为 PRISMA-ScR / scoping review
- [ ] 所有数据库使用同一主题逻辑、同一时间窗、同一天检索
- [ ] 检索式、字段范围、排除类型和时间过滤步骤可复现
- [ ] 同库去重与跨库去重规则写明且数字可追踪
- [ ] PRISMA 流程图与正文、表格、补充材料中的计数一致
- [ ] Results 同时给出命中统计与年度趋势，并说明统计口径
- [ ] 全文排除理由有审计记录
- [ ] 结论没有超出证据池边界
