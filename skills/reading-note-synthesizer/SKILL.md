# Reading Note Synthesizer

这个 Skill 负责把“读过很多论文，但信息散在高亮、便签、Markdown、会议记录里”的状态，转成可直接服务写作和决策的结构化资产。

它参考了热门科研工作流里最常见的一类能力: reading notes -> structured synthesis。但这里保持工具无关，只要求输入是你已经有的笔记、摘录、批注或手工摘要，不要求绑定 Zotero 或特定笔记软件。

## 何时调用

- 用户已经读过一批论文，但只有零散笔记
- 用户说“我有很多高亮/摘录，帮我整理成可写 related work 的材料”
- 用户要把 reading notes 变成问题卡、方法卡、局限卡
- 用户要为论文定位、讨论、审稿回应准备“文献弹药”
- 用户要把组会纪要、论文摘录、文献表整合成一份清晰综述底稿

## 核心目标

1. 把零散笔记转成结构化、可引用、可继续加工的研究资产
2. 保留原始出处与不确定性，不凭空脑补缺失内容
3. 支持后续 related work、positioning、claim checking、rebuttal
4. 优先生成“比较轴”和“局限轴”，而不只是摘要复述

## 顶会写作适配

这个 Skill 默认把笔记整理成 4 类最常用的投稿资产：

- `Related work ammo`
- `Positioning ammo`
- `Claim boundary notes`
- `Rebuttal ammo`

## 不该做什么

- 不把模糊印象写成确定结论
- 不把批注中的个人猜测误当成论文原文事实
- 不只做摘要拼接，必须整理出比较结构
- 不在材料缺失时硬凑差异点

## 推荐输入形态

以下任一形态都可以：

- 论文读书笔记
- PDF 高亮摘录
- Markdown 文献卡片
- 表格式论文比较记录
- 组会讨论纪要
- 你自己写的“这篇论文讲了什么 / 哪里不好 / 对我有什么用”

若材料来自 PDF 自动解析、OCR 或文档转 Markdown，额外建议保留：

- 页码或 section 锚点
- 原 PDF 中的表格 / 公式 / figure 对应位置
- 解析失败或疑似错位的片段标记

## 执行步骤

### Step 0: 检查输入是否带有解析噪声

若输入并非手工笔记，而是从 PDF、OCR 或自动解析工具得到，先筛出以下风险：

- 双栏串行导致句子顺序错乱
- 换行断词、连字符或脚注混入正文
- 表格被拍平成普通段落
- 公式、符号、上下标、希腊字母被破坏
- caption、图号、页眉页脚被误并入摘要或结论

这些片段先标记为 `解析风险片段`，不要直接当成事实摘录。

### Step 1: 先区分事实、判断和待核验项

把输入材料拆成三层：

- `事实摘录`：论文明确说了什么
- `个人判断`：你或合作者的评价、直觉、猜测
- `待核验项`：还需要回原文确认的位置

若输入来自 PDF 解析，再额外单列：

- `解析风险片段`：内容可能来自 OCR 错误、版面错位、表格扁平化或公式破坏

若没有这一步，后续很容易把个人印象写成 related work 事实。

### Step 2: 抽成研究卡片

至少整理以下卡片中的 3 到 5 类：

- `Problem Card`：论文处理什么问题，问题设置边界是什么
- `Method Card`：核心方法、机制、理论工具
- `Evidence Card`：理论、实验、benchmark、case study 各自提供了什么证据
- `Limitation Card`：作者自述局限和你观察到的局限
- `Usefulness Card`：对你当前论文最有帮助的点
- `Rebuttal Ammo Card`：可用于回应 reviewer 的对比与边界材料

### Step 3: 建比较轴

不要只按论文逐篇记录，要抽出比较维度，例如：

- 问题设定是否一致
- 方法是否依赖强假设
- 理论是否只覆盖简化情形
- 实验是否有真实规模验证
- 是否比较了最强 baseline
- 作者自己承认的局限是什么

### Step 4: 生成写作资产

根据用途输出不同资产：

- 如果要写 related work：输出分组、差异表、段落蓝图
- 如果要做定位：输出 gap 候选、不能乱说的边界
- 如果要做 rebuttal：输出 reviewer 可能争议点和对照证据
- 如果要做 claim 检查：输出哪些文献能支撑、哪些只能弱支撑

### Step 4.5: 组织成可直接 handoff 的素材包

每份素材包尽量都包含：

- `可直接复用内容`：已经足够清晰，可直接交给下游 Skill
- `需保守表达内容`：能写，但必须降级口径
- `需回原文核验内容`：暂不能直接写进论文
- `不建议使用内容`：材料太弱、来源不稳或容易误导

### Step 5: 标出需要回原文核验的地方

凡是以下情况都应单列：

- 只来自二手笔记，没有原句定位
- 你的评价和论文原文可能不完全一致
- 论文结论容易被过度泛化
- 摘录信息不足以支持强比较
- 摘录来自 PDF 解析，且关键信息涉及表格、公式、figure caption 或多栏排版

## 默认输出格式

- `材料概况`
- `事实 / 判断 / 待核验项`
- `研究卡片`
- `比较轴`
- `写作可用资产`
- `仍需补查的问题`

## 推荐输出模板

### 模板 A: Related Work 素材包

- `主题簇`
- `代表论文`
- `比较轴`
- `各路线局限`
- `可直接展开成段落的蓝图`

### 模板 B: Positioning 素材包

- `现有工作已覆盖什么`
- `现有工作没解决什么`
- `哪些差异是真差异`
- `哪些说法不要写太满`

### 模板 C: Rebuttal 素材包

- `潜在 reviewer concern`
- `现有文献证据`
- `可承认的局限`
- `可反驳的误解`

## 默认保守策略

- 若笔记没有原文出处，默认只当线索，不当最终证据
- 若你的评价和作者自述冲突，优先保留冲突并标记待核验
- 若摘录无法支撑横向比较，默认先拆出待补查项
- 若一篇论文只是背景相关，不强行纳入主比较表
- 若笔记底层来自 PDF 自动解析结果，默认不把公式、表格数字或 caption 细节当成已核验事实

## 标准输入

- 一组阅读笔记、摘录、批注、会议记录或论文卡片
- 本轮目标：related work / positioning / rebuttal / claim support
- 是否保留原文出处定位
- 是否允许把内容整理成表格或卡片
- 若输入来自 PDF / OCR / 自动解析，是否已保留页码锚点与原 PDF 对照位置

## 标准交付

- 结构化研究卡片
- 比较轴与差异表
- 可直接交给下游 Skill 的素材包
- 需要回原文补查的风险点
- 若需要，附 `Related Work / Positioning / Rebuttal / Claim boundary` 四类素材包之一
- 若涉及 PDF 解析输入，附 `Parsing Noise / Must Recheck in Original PDF`

## Examples

- “这些论文我都读过了，帮我把读书笔记整理成 related work 资产。”
- “把这批高亮和组会讨论记录整理成 gap 和 limitation 清单。”
- “我不需要再摘要一遍，重点帮我抽出可用于 rebuttal 的文献证据。”
- “把这些 notes 转成 problem-method-evidence-limitation 四列卡片。”
